# PP（Promenade Position）設計書 — 社交ダンス フィガー学習アプリ

作成日: 2026-06-11
ステータス: 承認待ち

## 1. 概要と目的

社交ダンスの足型・フィガーを学習するためのウェブアプリ。フィガー名と足型の対応を覚えることを最優先とし、足の位置関係・足先の角度・体の向き・重心（トー/ヒール/フラット）を視覚的に分かりやすく表示する。

- **利用者**: まず開発者本人の個人練習用。将来、本人のダンス関連サイトで一般公開する
- **対象種目**: スタンダード＋ラテン全10種目を視野に、初版はワルツの基本フィガー（10個前後）
- **内容のベース**: ISTD系教本（「ボールルームダンステクニック」等）
- **データ作成**: 足型データはClaudeが全量生成。利用者が教本と照合し、相違があれば会話で修正を依頼する（アプリ内エディタは作らない）

## 2. 技術方針

- **スタック**: Vite + React + TypeScript の静的SPA
- **描画**: SVG（足型図・アニメーションとも）
- **データ**: 静的JSONファイル（サーバー・DB不要）
- **配布**: `vite build`（`base: './'` の相対パス設定）で生成した静的ファイル一式を、任意のサイトのサブディレクトリに置くだけで動作する
- **ルーティング**: ハッシュ形式（`#/waltz/natural-turn`）。サーバー側のリライト設定が不要なため
- **対応デバイス**: スマホ・PC両対応のレスポンシブ
- **多言語対応（i18n）**: 全10言語に切替可能な設計。日本語・英語・韓国語・タガログ語・簡体字中国語・繁体字中国語（台湾）・ポルトガル語・スペイン語・イタリア語・フランス語

### 多言語化の仕組み

- **UI文言**: 言語ごとの辞書JSON（`src/locales/ja.json`, `en.json`…）。ヘッダーの言語スイッチャーで切替、選択はlocalStorageに保存
- **ISTD用語はコード値で保持**: アライメント・フットワーク・ライズ&フォール・スウェイ・足の位置などは決まった語彙の組み合わせなので、フィガーデータにはコード値（例: `facing_DW`）を格納し、表示時に言語辞書で変換する。これによりフィガーデータは1セットのまま全言語に対応できる
- **フィガー名**: 言語別オブジェクト（`"name": { "ja": "ナチュラルターン", "en": "Natural Turn", ... }`）。未翻訳の言語は英語にフォールバック（ISTD用語の原語は英語のため）
- **自由記述の`note`**: 言語別オブジェクト。存在しない言語は英語→日本語の順でフォールバック

## 3. 画面構成（3画面）

1. **種目選択** — ワルツ、タンゴ…のリスト
2. **フィガー一覧** — 選択した種目のフィガー目次（名前・英名・歩数・レベル）
3. **フィガー詳細** — メイン画面。以下で構成:
   - **役割切替**（男性/女性/両方）: 常時表示。「両方」は2人の足を同一フロア座標に重ね、位置関係（コンタクトポジション）を示す
   - **フロア図（SVG俯瞰足型図）**: 教本風に全歩を1枚で俯瞰。LOD（進行方向）矢印・床グリッド付き
   - **再生バー**: ▶でアニメーション再生（カウントに合わせ足型が移動）。一時停止・0.5x/1x/2x速度
   - **歩ナビ**: スマホ=歩番号チップ（横並び）、PC=ISTD表（全歩・全列）。図とクリックで相互連動
   - **歩詳細パネル**: 選択中の歩のISTDデータを表示

### 足型図の表現ルール

- **重心の色分け**: トー=赤系(#e76f51)、ヒール=青系(#457b9d)、フラット(全体)=緑系(#2a9d8f)、体重なし=点線輪郭。足の前部（ボール/トー）と後部（ヒール）を別パーツとして塗り分ける
- **左右の判別（3段構え）**:
  1. 形 — 親指側がふくらみ土踏まず側がくびれた非対称シルエット（左右でミラー）
  2. ラベル — かかと部に L/R を常時表示、歩番号はつま先側
  3. 輪郭色 — 左足=青系輪郭、右足=赤茶系輪郭
- **「両方」表示時のペア区別**: 男性=濃色系、女性=淡色系（実装時に微調整）
- **体の向き**: 矢印インジケーターで表示
- 視覚化するのは空間的な情報（位置・角度・重心・体の向き）。カウント・回転量・アライメント・ライズ&フォール・スウェイ・CBMはテキスト（詳細パネル/表）で正確に示す。スウェイの図示（体の傾き表現）は将来拡張とする

## 4. データ構造

1フィガー=1JSONファイル。アプリ本体とデータを完全分離する。

```
public/data/
├── dances.json            # 種目一覧
└── waltz/
    ├── figures.json       # ワルツのフィガー目次
    ├── natural-turn.json  # フィガー詳細
    └── ...
```

### フィガー詳細JSON

```json
{
  "id": "natural-turn",
  "name": { "ja": "ナチュラルターン", "en": "Natural Turn" },
  "dance": "waltz",
  "timeSignature": "3/4",
  "parts": {
    "man":  { "steps": [ ... ] },
    "lady": { "steps": [ ... ] }
  }
}
```

### 1歩分（ISTDチャートの全項目＋描画用座標）

ISTD項目はすべてコード値で保持し、表示時に言語辞書で各言語の文言に変換する（§2 多言語化の仕組み参照）。

```json
{
  "stepNo": 2,
  "foot": "L",
  "stepDescription": { "move": "side", "modifier": "slightly_back" },
  "count": "2",
  "footwork": "T",
  "alignment": { "relation": "backing", "direction": "DW" },
  "amountOfTurn": { "direction": "right", "amount": "1/4", "between": [1, 2] },
  "riseAndFall": "continue_rise",
  "sway": "R",
  "cbm": false,
  "position": { "x": 40, "y": 55, "angle": 135 },
  "note": {}
}
```

- `footwork` が重心色の源。`H`/`T`/`TH`/`HT` 等を解析して前部・後部を塗り分け、順序（T→H等）は詳細パネルに表示
- `alignment` は関係（facing/backing/pointing）＋方向（LOD/DW/DC/壁/中央 等）の組み合わせコード。`stepDescription`・`riseAndFall`・`sway` も同様に定義済みコードのみ許可し、検証スクリプトでチェックする
- `position` はフロア共通座標（x, y）と足の向き（angle, 度）。男性・女性は同一座標系のため重ね表示がそのまま成立する
- アニメーションは歩ごとの座標列をカウントのタイミングで補間して自動生成（アニメ専用データは持たない）
- 未知の記号・欠損値はビルド前の検証スクリプトで検出する

## 5. コンポーネント構成

```
src/
├── types.ts              # フィガー/ステップの型定義（ISTDコード値の列挙型を含む）
├── data/loader.ts        # JSON読み込み＋形式チェック
├── i18n/                 # 言語辞書と変換ロジック
│   ├── index.ts          # useTranslation相当のフック・コード値→文言変換
│   └── locales/          # ja.json, en.json, ko.json, tl.json, zh-CN.json,
│                         #   zh-TW.json, pt.json, es.json, it.json, fr.json
├── hooks/useAnimation.ts # 再生・一時停止・速度・補間
├── pages/
│   ├── DanceList.tsx
│   ├── FigureList.tsx
│   └── FigureDetail.tsx
└── components/
    ├── FloorDiagram.tsx  # SVGフロア図
    ├── Foot.tsx          # 足1つ（非対称形・重心色・L/R・角度）
    ├── RoleToggle.tsx
    ├── LanguageSwitcher.tsx # 言語切替（選択をlocalStorageに保存）
    ├── PlaybackBar.tsx
    ├── StepTable.tsx     # PC=ISTD表 / スマホ=歩チップ
    └── StepDetailPanel.tsx
```

i18nライブラリは導入せず、軽量な自前実装とする（辞書JSON＋Reactコンテキスト）。理由: 必要なのは「キー→文言」と「コード値→文言」の変換のみで、複数形処理などの高度な機能は不要なため。

## 6. エラー処理

- データ読み込み失敗 → エラーメッセージ＋再試行ボタン（アプリは落とさない）
- データ値の誤り（未知のフットワーク記号等）→ ビルド前検証スクリプト（`npm run validate-data`）で検出
- 実行時に未知の値に遭遇した場合 → 該当の足をグレー表示にして描画を継続

## 7. テスト

- **Vitest単体テスト**: フットワーク記号→色マッピング、アニメ補間計算、データ検証ロジック
- **データ検証**: 全JSONをビルド時に型・値チェック
- **内容の最終確認**: 足型図と教本の照合は利用者がブラウザで実施（Claudeの生成データは参考値であり教本が正）

## 9. リスクと対策

- **生成データの正確性**: ClaudeのISTDデータは誤りを含みうる → 教本照合を運用に組み込み、修正は会話で依頼
- **アニメーションの複雑さ**: 補間が不自然になる可能性 → v1は単純な直線＋回転補間から始め、必要なら改善
- **「両方」表示の足の重なり**: コンタクトポジションでは足が近接して見づらい可能性 → 透過度・オフセット等で実装時に調整
