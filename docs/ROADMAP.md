## 最優先: フィガーデータの教本照合（ユーザー作業＋会話で修正）

データはClaude生成の参考値。ISTD教本と見比べて、違う箇所を会話で指摘 → 修正してプッシュ、の運用。
生成時に自信が低かった箇所（優先的に確認）:

1. **ナチュラルターンの終了方向** — 現データはDC（中央斜め）。教本がDWなら戻す
2. **アウトサイドチェンジ** — 入りのアライメント（現: 中央斜めに背面）とスウェイ（現: 全歩ストレート）、CBM（現: 2歩目のみ）
3. **クローズドチェンジ** — アライメント（N→R: 中央斜めに面したまま回転なし）、スウェイ S,R,R / S,L,L
4. **リバースコルテ男性** — 3歩目を「体重を右足へ戻す（TH）」とモデル化。2歩目 T・わずかにライズ
5. **ヘジテーションチェンジ** — 男性4歩目 TH、5歩目 フラット＋ノート、4-6歩 ライズ＆フォールなし
6. **ナチュラルスピンターン** — 男性4歩目 THT、5歩目 HT・2の終わりにライズ、CBM配置
7. **ウィスク女性2歩目** — 中央斜めにポイント・1-2間1/4
8. **シャッセ フロム PP** — 女性の左回転を1-2間/2-3間に各1/8と配分、スウェイ全歩ストレート
9. **座標の見た目** — ターン系3歩目のペア間隔がやや広い（32単位）など、図で不自然な箇所は随時調整

## 機能拡張の候補（おすすめ順）

### 短期（学習効果が高い・作りやすい）
1. **クイズ・暗記カードモード** — フィガー名⇔足型の一致テスト。ブレスト時の「1〜5すべてほしい」のうち未実装の本命。例: 足型図を見て名前を4択で当てる／名前からカウント・歩数を答える
2. **習得チェック・お気に入り** — フィガーごとに「覚えた/練習中」マーク（localStorage保存）。一覧でフィルタ
3. **選択した歩へのズーム** — フォーカス表示の強化。図が混み合うフィガーで効く
4. **カウント表示の強化** — アニメ再生中に「1, 2, 3」を大きく表示（現在は足の動きのみ）

### 中期（コンテンツ拡大）
5. **残り8言語の辞書** — 韓国語・タガログ語・簡体字/繁体字中国語・ポルトガル語・スペイン語・イタリア語・フランス語。辞書ファイル追加のみで対応可能な設計済み
6. **他種目の追加** — タンゴ→スタンダード残り→ラテン。データ追加だけでアプリ側の変更は不要（dances.json の available を true に）
7. **アマルガメーション（フィガー連結）再生** — 複数フィガーをつなげたルーティンを通しで再生。練習の流れの確認に
8. **スウェイ・ライズの視覚化** — 体の傾きや上下動を図で表現（v1で意図的に見送った項目）

### 長期（公開・運用まわり）
9. **PWA化** — ホーム画面追加＋オフライン動作。練習場の電波が弱くても使える
10. **ダンスサイトへの統合** — `dist/` をサイトのサブディレクトリにコピーするだけで動く設計済み。デザインをサイトに合わせる場合は frontend-design スキルで調整
11. **メトロノーム/音楽同期** — カウント音やワルツ曲に合わせた再生

## 技術メモ（次のセッション用）

- 開発: `npm run dev` ／ 検証: `npm test`（全フィガーデータのスキーマ・値チェック込み）／ データだけ: `npm run validate-data`
- デプロイ: main にプッシュするだけ（.github/workflows/deploy.yml）。ビルドはテストがゲート
- フィガー追加手順: ①`public/data/waltz/<id>.json` を natural-turn.json を雛形に作成 ②`figures.json` に目次追加 ③`npm test` ④コミット&プッシュ
- ISTD項目はすべてコード値（src/types.ts の列挙）。新しい値が必要なら列挙に追加＋ja/en両辞書に対訳追加（型が漏れを検出）
- 座標規約: LOD=+x（画面右）、angle 0=画面上向き・時計回り、足の描画長約25単位、隣接歩25-40単位・クローズ10-15単位
- 設計書: `docs/superpowers/specs/2026-06-11-pp-dance-figures-design.md` ／ v1実装計画: `docs/superpowers/plans/2026-06-11-pp-v1.md`
